## 二、P1-严重问题（1个）

### P1-01: 5个B3设备控制SIPp场景XML根元素错误

- **文件**：
  - `tests/sipp/control/ctrl_b3_01_ptz.xml`
  - `tests/sipp/control/ctrl_b3_05_teleboot.xml`
  - `tests/sipp/control/ctrl_b3_08_record.xml`
  - `tests/sipp/control/ctrl_b3_10_home_position.xml`
  - `tests/sipp/control/ctrl_b3_12_i_frame.xml`
- **审计方法**：#106 XML根元素(Control vs Notify)、#91 规范合规性
- **问题描述**：
  以上5个设备控制测试场景的XML根元素使用 `<Notify>`，但GB/T 28181规范中：
  - 设备控制命令的XML根元素应为 `<Control>`
  - 设备通知的XML根元素为 `<Notify>`
  
  注意：`ctrl_snapshot_finished_notify.xml` 和 `ctrl_upgrade_result_notify.xml` 使用 `<Notify>` 是正确的（它们是通知消息）。
- **影响**：5个设备控制测试场景XML格式不符合规范，真实设备可能拒绝处理。测试"通过"只是因为模拟器返回200 OK。
- **修复建议**：将这5个场景的 `<Notify>` 改为 `<Control>`，`</Notify>` 改为 `</Control>`。

## 三、P2-一般问题（4个）

### P2-01: 补丁02中MD5摘要条件判断使用equals（时序攻击风险）

- **文件**：`patches/02-DigestServerAuthenticationHelper.patch` 第59行
- **审计方法**：#88 时序攻击防护
- **问题描述**：
  ```java
  if (!mdString.equals(response) && sm3 != null) {
  ```
  此处用 `equals` 判断MD5摘要是否匹配以决定是否尝试SM3回退。虽然最终认证比较（第82行）已改为 `MessageDigest.isEqual`，但条件判断中的 `equals` 仍有时序攻击风险。
- **影响**：攻击者可通过时序分析判断MD5是否匹配，辅助密码破解。风险较低（仅影响回退判断，不影响最终认证）。
- **修复建议**：改为 `!MessageDigest.isEqual(mdString.getBytes(StandardCharsets.UTF_8), response.getBytes(StandardCharsets.UTF_8))`。

### P2-02: merge.sh 颜色码不完整

- **文件**：`scripts/merge.sh` 第12行
- **审计方法**：#19 Shell脚本语法
- **问题描述**：
  ```bash
  RED='\033'; GREEN='\033'; YELLOW='\033'; NC='\033'
  ```
  颜色码只写了 `\033`（ESC字符），缺少颜色码后缀。正确应为 `RED='\033[0;31m'` 等。
  当前所有颜色输出都是空转（ESC字符后直接跟文本，终端不会着色）。
- **影响**：日志输出无颜色区分，不影响功能但影响可读性。
- **修复建议**：
  ```bash
  RED='\033[0;31m'; GREEN='\033[0;32m'; YELLOW='\033[1;33m'; NC='\033[0m'
  ```

### P2-03: frontEnd.js ptzPrecise默认值0是有效值

- **文件**：`ui/api/frontEnd.js` ptzPrecise函数
- **审计方法**：#82 输入验证
- **问题描述**：
  ```javascript
  params: { pan: pan ?? 0, tilt: tilt ?? 0, zoom: zoom ?? 1 },
  ```
  `pan ?? 0` 中 0 是有效的水平转角（表示正前方）。当调用方未传 pan 参数时，默认为 0 会导致云台误动到正前方。
- **影响**：调用方忘记传参数时，云台可能误动。
- **修复建议**：默认值改为 null，发送请求前校验非null：
  ```javascript
  params: { pan: pan ?? null, tilt: tilt ?? null, zoom: zoom ?? null },
  ```
  并在函数开头校验：`if (pan == null || tilt == null || zoom == null) throw new Error('pan/tilt/zoom为必填')`

### P2-04: SnapshotConfigMessageHandler 仍有TODO

- **文件**：`news/main/java/com/genersoft/iot/vmp/gb28181/transmit/event/request/impl/message/control/cmd/SnapshotConfigMessageHandler.java` 第265行
