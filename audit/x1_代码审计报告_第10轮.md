## 二、P1-严重问题（新增）

### P1-24: UserSetting配置项未与实际代码关联 — 配置不生效

**文件**: 
- 补丁: `patches/06-UserSetting.patch`
- 代码: `news/main/java/com/genersoft/iot/vmp/gb28181/utils/*.java`

**问题**: UserSetting.patch定义了20+个配置项，但实际代码使用`System.getProperty()`读取配置，而非通过UserSetting注入。配置项定义与代码使用完全脱节。

**补丁文档声明** (第14行):
```java
// 审计修复R2-P1-01: 以下配置项需由各组件通过UserSetting注入读取, 确保配置生效
private boolean gb28181Version2022Enabled = true;
private boolean sm3DigestEnabled = true;
private boolean registerRedirectEnabled = true;
// ... 20+个配置项
```

**实际代码使用** (各工具类):
```java
// SM3DigestHelper.java
System.getProperty("wvp.sm3.digest.enabled", "true")

// MansrtspHelper.java
System.getProperty("wvp.mansrtsp.scale.limit.enabled", "true")

// SdpFieldHelper.java
System.getProperty("wvp.sdp.2022fields.enabled", "true")

// GBProtocolVersionHelper.java
System.getProperty("wvp.xgbver.header.enabled", "true")
```

**影响**: 
1. **配置不生效**: 在`application.yml`中配置`user-setting.sm3-digest-enabled=false`不会生效，因为代码读取的是JVM系统属性
2. **配置项浪费**: UserSetting中定义的20+个配置项全部无效，增加维护负担
3. **审计修复未落实**: 补丁注释声称"审计修复R2-P1-01"，但实际代码并未使用UserSetting注入
4. **运维困难**: 运维人员需要通过`-Dwvp.sm3.digest.enabled=false`JVM参数配置，而非Spring配置文件

**修复建议**: 
1. 将各工具类改为Spring Bean，通过`@Autowired UserSetting`注入配置
2. 或将工具类的静态方法改为实例方法，通过Spring注入UserSetting
3. 移除`System.getProperty()`调用，统一使用UserSetting配置

## 三、P2-中等问题（新增）

### P2-58: UserSetting中dataIntegrityEnabled和gb35114CertAuthEnabled默认为false，但相关功能是桩实现

**文件**: `patches/06-UserSetting.patch`

**问题**: 
- 第143行: `private boolean dataIntegrityEnabled = false;` — 数据完整性默认禁用
- 第149行: `private boolean gb35114CertAuthEnabled = false;` — GB35114证书认证默认禁用

这两个配置项默认为false（禁用），但即使启用，相关功能也是桩实现（P1-17, P1-18, P1-19），无法实际工作。

**影响**: 
- 运维人员启用这些配置后，会发现功能仍不可用
- 可能产生误导：以为启用了安全功能，实际未生效

**修复建议**: 
1. 在配置项注释中明确标注"当前为桩实现，启用后功能不可用"
2. 或在功能完整实现前，将配置项移除或标记为`@Deprecated`
