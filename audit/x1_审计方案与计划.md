## 三、审计执行流程

### 3.1 多轮迭代审计

```
Round 1: 全量扫描 → 记录所有发现的问题
Round 2: 针对Round 1发现的问题进行深度分析 → 记录新发现
Round 3: 聚焦线程安全与并发问题 → 记录新发现
Round 4: 聚焦安全漏洞 → 记录新发现
Round 5: 聚焦GB28181-2022规范合规性 → 记录新发现
Round 6: 聚焦可靠性与健壮性 → 记录新发现
Round 7: 聚焦资源管理与泄漏 → 记录新发现
Round 8: 聚焦测试质量 → 记录新发现
Round 9: 聚焦代码质量 → 记录新发现
Round 10: 聚焦文档一致性 → 记录新发现
...
Round N: 持续迭代直到连续50轮无新增问题
```

### 3.2 问题分级标准

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0-致命** | 代码无法编译/运行，或存在直接安全漏洞 | 编译错误、认证绕过 |
| **P1-严重** | 生产环境中会导致功能失效或数据损坏 | 线程安全、资源泄漏 |
| **P2-中等** | 在特定条件下会导致问题 | 空指针、超时缺失 |
| **P3-轻微** | 代码质量问题，不影响功能 | 命名不规范、注释缺失 |
