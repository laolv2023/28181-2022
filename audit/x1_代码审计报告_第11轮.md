## 二、P2-中等问题（新增）

### P2-59: SnapshotConfigDialog.vue — abortController声明但未在API请求中使用

**文件**: `ui/components/SnapshotConfigDialog.vue`

**问题**: `abortController`在data中声明，在`beforeDestroy`中清理，但在`handleSnapshot`方法中从未创建或传递给API请求。

**代码位置**: 
- 第137行: `abortController: null` (声明)
- 第176行: `if (this.abortController) { this.abortController.abort(); }` (清理)
- 第207-236行: `handleSnapshot`方法中未创建abortController

**分析**: 
- `abortController`始终为null
- `beforeDestroy`中的清理是空操作
- 如果组件在API请求期间被销毁，请求不会被取消
- 与P1-16（QueryResultDialog.vue）同类问题，但严重程度较低（单次POST请求）

**影响**: 
- 组件销毁时API请求不会被取消
- 可能导致已销毁组件的状态被更新（Vue警告）

**修复建议**: 在`handleSnapshot`中创建AbortController并传递给API请求：
```javascript
this.abortController = new AbortController()
await snapshotConfig({ ..., signal: this.abortController.signal })
```

### P2-60: SnapshotConfigDialog.vue — snapshotList的index字段在删除旧记录后可能重复

**文件**: `ui/components/SnapshotConfigDialog.vue`

**问题**: 第222-223行，`index`使用`snapshotList.length + 1`计算，但当列表超过20条时执行`slice(0, 20)`截断，后续新记录的index可能与已有记录重复。

**代码位置**: 第222-230行
```javascript
this.snapshotList.unshift({
    index: this.snapshotList.length + 1,  // 基于当前长度+1
    fileId: sessionId,
    timestamp: new Date().toLocaleString()
})
if (this.snapshotList.length > 20) {
    this.snapshotList = this.snapshotList.slice(0, 20)  // 截断
}
```

**分析**: 
- 假设当前列表有20条记录（index 1~20）
- 新增第21条：`index = 20 + 1 = 21`，unshift后列表变为21条
- `slice(0, 20)`截断后列表恢复20条，但index为21的记录在第一位
- 新增第22条：`index = 20 + 1 = 21`，与第一条记录的index重复

**影响**: 
- 抓拍记录序号重复，用户困惑
- 如果以index为key，可能导致Vue渲染异常

**修复建议**: 使用递增计数器或时间戳作为index：
```javascript
data() { return { snapshotCounter: 0 } }
// ...
this.snapshotCounter++
this.snapshotList.unshift({ index: this.snapshotCounter, ... })
```

## 三、P3-轻微问题（新增）

### P3-34: DeviceUpgradeDialog.vue — 进度条百分比计算可能超过100%

**文件**: `ui/components/DeviceUpgradeDialog.vue`

**问题**: 升级进度通过轮询获取，如果设备报告的进度值异常（如105%），进度条可能显示超过100%。

**分析**: 
- Element UI的`el-progress`组件在`percentage`超过100时会显示异常
- 应对进度值进行边界检查：`Math.min(100, Math.max(0, progress))`

**修复建议**: 在设置进度时添加边界检查。
