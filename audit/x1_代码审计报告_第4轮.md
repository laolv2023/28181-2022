## 二、P1-严重问题（新增）

### P1-21: MansrtspHelper.java — appendScale方法在builder为null时抛NPE

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/utils/MansrtspHelper.java`

**问题**: `appendScale`方法在`builder`为`null`且`wvp.mansrtsp.scale.limit.enabled`为`false`时，直接调用`builder.append(...)`导致NullPointerException。

**代码位置**: 第71行

```java
// 第71行: 当scale限制被禁用时，直接使用builder，不检查null
if (!Boolean.parseBoolean(System.getProperty("wvp.mansrtsp.scale.limit.enabled", "true"))) 
    { builder.append("Scale: 1.0\r\n"); return DEFAULT_SCALE; }  // NPE if builder==null

// 第72-74行: 当scale限制启用时，检查builder==null
if (builder == null) {
    return DEFAULT_SCALE;
}
```

**影响**: 
- 当`wvp.mansrtsp.scale.limit.enabled=false`且`builder=null`时，抛出NullPointerException
- 与scale限制启用时的行为不一致（启用时返回DEFAULT_SCALE，禁用时抛NPE）

**修复建议**: 在第71行之前添加`builder == null`检查，统一行为。

## 三、P2-中等问题（新增）

### P2-30: TcpReconnectHelper.java — 重复的最小间隔检查

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/utils/TcpReconnectHelper.java`

**问题**: `reconnectTcpMedia`方法中存在两处最小间隔检查：
- 第92-94行: 检查 `intervalMs < DEFAULT_INTERVAL_MS` 并修正
- 第105行: 再次检查 `intervalMs < 1000` 并修正

如果 `DEFAULT_INTERVAL_MS` 不等于 1000，两处检查可能产生不一致行为。

**修复建议**: 统一使用一个常量进行最小间隔检查。

### P2-31: TcpReconnectHelper.java — reconnectTcpMedia方法没有取消机制

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/utils/TcpReconnectHelper.java`

**问题**: 重连循环没有提供取消机制，应用关闭时无法中断正在进行的重连。

**影响**: 
- 应用关闭时重连线程可能继续运行，导致资源泄漏
- 无法在运行时取消重连

**修复建议**: 添加`volatile boolean cancelled`标志或`Thread.interrupt()`检查。

### P2-32: ExtensionApplicationHandler.java — 空实现，所有扩展应用消息被静默接受

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/utils/ExtensionApplicationHandler.java`

**问题**: `handleExtensionApplication`方法对所有非空XML内容返回`true`，不做任何处理。

**代码位置**: 第36行

```java
public boolean handleExtensionApplication(String xmlContent) {
    if (xmlContent == null || xmlContent.trim().isEmpty()) {
        return false;
    }
    return true; // 接收并记录, 实际处理需根据业务实现
}
```

**影响**: 
- 所有扩展应用消息被"处理成功"，但实际什么都没做
- 注释说"接收并记录"，但没有实际的日志记录
- 调用方误以为消息已处理

**修复建议**: 至少添加日志记录，或返回`false`表示未实现。

### 5.2 关键发现

1. **MansrtspHelper空指针风险（P1-21）**: `appendScale`方法在特定配置下抛出NPE，与正常路径行为不一致。

2. **扩展应用处理器空实现（P2-32）**: 所有扩展应用消息被静默接受，不做任何处理，调用方误以为已处理。

3. **SDP字段命名注释矛盾（P2-33）**: 注释中"DoWnload"和"Download"混用，"首字母大写"描述与实际全小写代码矛盾，表明开发过程中存在命名混淆。

4. **GB编码校验范围不完整（P2-35）**: 行政区码省级码段缺少台湾(71)、香港(81)、澳门(82)。

5. **TCP重连无取消机制（P2-31）**: 应用关闭时无法中断重连，可能导致资源泄漏。
