## 二、P2-一般问题（新增2个）

### R5-P2-01: SipMessageFilter读取消息体异常仅DEBUG级别记录

- **文件**：`news/main/java/com/genersoft/iot/vmp/gb28181/utils/SipMessageFilter.java` 第159-161行
- **审计方法**：#6 异常吞没、#97 日志规范
- **问题描述**：
  ```java
  } catch (Throwable t) {
      logger.debug("[SIP过滤] 读取消息体异常: {}", t.getMessage());
  }
  ```
  读取消息体时捕获 `Throwable`（包括 `OutOfMemoryError`——可能因超大消息体触发），但仅以 `DEBUG` 级别记录。生产环境通常禁用 DEBUG 日志，此异常会被静默吞没。如果消息体读取反复失败（如网络问题或恶意超大消息），运维人员无法从日志中发现。
- **影响**：生产环境消息体读取异常无法被监控发现，可能导致SIP消息处理静默失败。
- **修复建议**：提升为 `WARN` 级别，并考虑对 `OutOfMemoryError` 等严重错误单独处理。

### R5-P2-02: SipMessageFilter X-GB-ver头部解析异常仅DEBUG级别记录

- **文件**：`news/main/java/com/genersoft/iot/vmp/gb28181/utils/SipMessageFilter.java` 第170-172行
- **审计方法**：#6 异常吞没、#97 日志规范、#85 协议版本协商
- **问题描述**：
  ```java
  } catch (Throwable t) {
      logger.debug("[SIP过滤] X-GB-ver 头部解析异常: {}", t.getMessage());
  }
  ```
  X-GB-ver 头域解析异常仅以 `DEBUG` 级别记录。X-GB-ver 是2022版协议版本协商的关键头域，解析失败意味着无法确定设备协议版本，可能导致2022功能错误启用/禁用。生产环境此异常被静默吞没。
- **影响**：协议版本协商异常无法被监控发现，可能导致版本判断错误。
- **修复建议**：提升为 `WARN` 级别。
