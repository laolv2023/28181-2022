## P2 (Medium)

### R7-P2-001: SdpFieldHelper.java — buildSFieldForTalk 返回空字符串但调用方可能未处理

**文件**：`news/.../SdpFieldHelper.java` L113-116
**维度**：E06
**问题**：
```java
public static String buildSFieldForTalk() {
    // 改造项5：2022版取消 Talk 类型支持，s 字段不再写入 Talk
    return "";
}
```
返回空字符串。如果调用方直接将返回值拼入 SDP `s=` 行，会产生 `s=\r\n`（空 s 值），某些 SIP 实现可能拒绝。SDP 规范要求 `s` 字段非空（建议 "-" 或 "Play"）。

**修复建议**：返回 `null` 并让调用方跳过写入，或返回 `"-"`。

### R7-P2-002: GbCode2022.java — isValidCapturePositionCode 使用 2 位子串但设计文档说 1 位

**文件**：`news/.../GbCode2022.java` L285-288
**维度**：E17
**问题**：
```java
// 审计修复P2-13: 采集位置码取 2 位以支持 1~10 范围, 确保边界值 10 可被校验通过
String capturePositionCode = gbCode.substring(13, 15);
```
设计文档第 13.5 节说"采集位置码：1~10 共 10 种取值"。如果采集位置码是编码中的 1 位（第 14 位），取值 1~9 是单数字，10 需要 2 位。但编码结构中每个字段位置固定，取 2 位可能侵占了后续字段。

需查证 2022 规范附录 E 编码结构确认采集位置码的字段宽度。

**影响**：如果规范定义采集位置码为 1 位（取值 1~9），则 10 不合法，取 2 位会导致后续字段偏移。
**修复建议**：查证规范原文确认字段宽度。
