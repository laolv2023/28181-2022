## 三、关键发现

### 3.1 P0 级问题（必须修复）

1. **3 个 auth 接口文件存在编译错误**：`CertAuthHelper.java`、`DataIntegrityHelper.java`、`GB35114Helper.java` 在 Javadoc `*/` 后有 stray `*` 行，导致 Java 编译失败。

2. **3 个 XML 构造方法存在注入风险**：`ptzPreciseCmdImpl`、`storageCardFormatCmdImpl`、`deviceUpgradeCmdImpl` 直接拼接外部输入到 XML，无转义处理。

3. **前端 SecurityConfig.vue 方法未定义**：`loadConfig` 方法在 `created` 中被调用但未实现。

4. **前后端 API 参数传递不一致**：前端使用 `data`（JSON body），后端使用 `@RequestParam`（form-data）。

### 3.2 P1 级问题（建议上线前修复）

1. **SM3 回退到 MD5 时无降级标识**：安全级别下降但系统无感知。
2. **SM3DigestHelper Provider 初始化非线程安全**。
3. **设备升级文件上传缺少大小限制和路径校验**。
4. **PTZ 精准控制参数校验不足**。
5. **前后端 HTTP 方法不一致**（GET vs POST）。

### 3.3 P2 级问题（建议后续迭代修复）

1. **配置不持久化**（ApiConfigController 内存存储）。
2. **密码以 String 存储**（SipTlsProperties）。
3. **代码重复**（多个 Handler 的 responseAck 方法）。
4. **Locale 依赖的浮点格式化**。
5. **补丁文件缺少精确定位**。
