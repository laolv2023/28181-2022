## 二、P0-致命问题（4个）

### P0-01: GB35114HelperImpl.java 编译错误（return后语法错误）

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/auth/impl/GB35114HelperImpl.java`
**行号**: 28-31
**问题**: `checkSecurityLevel` 方法中 `return` 语句后直接跟注释和代码，缺少分号，导致编译错误。

**代码片段**:
```java
@Override
public boolean checkSecurityLevel(String declaredLevel, String requiredLevel) throws GB35114Exception {
    if (declaredLevel == null || requiredLevel == null) {
        throw new GB35114Exception("安全级别参数不能为null");
    }
    // 简单比较: 声明级别 >= 要求级别
    return // 安全级别比较: A < B < C < D       // ← 错误：return后无表达式
    int declaredRank = "ABCD".indexOf(declaredLevel);  // ← 编译错误
    int requiredRank = "ABCD".indexOf(requiredLevel);
    return declaredRank >= 0 && declaredRank >= requiredRank;  // ← 重复return
}
```

**影响**: 项目无法编译，GB35114高安全级别认证功能完全不可用。
**修复建议**: 将第28行改为 `// 安全级别比较: A < B < C < D` 注释单独一行，删除多余的 `return`。

## 九、系统性问题总结

本次审计发现的最严重系统性问题是：**GB/T 28181-2022规范要求的XML元素名大小写与代码实现不一致**。具体表现为：

| 元素名 | 规范要求 | 代码实现 | 影响 |
|--------|---------|---------|------|
| s字段 | `DoWnload`（W大写） | `Download`（小写w） | 测试A3.1失败 |
| downloadspeed | `doWnloadspeed`（W大写） | `downloadspeed`（全小写） | 测试A3.3失败 |
| FileuRL | `FileuRL`（小写u） | `FileURL`（大写U） | 测试A4.4失败 |
| PTZ pan/zoom | 需统一 | 大小写不一致 | 解析异常 |

**结论**: 在38项改造中，约29%（11/38）存在"有代码无集成"的问题，约11%（4/38）存在合规性大小写bug。**建议在发布前逐一验证各改造项的端到端功能，特别是XML元素名大小写规范**。
