## 一、审计摘要

### 1.1 审计范围

本次审计对《GB/T 28181—2016 升级至 GB/T 28181—2022 开发设计文档》（以下简称"设计文档"）进行了全面、逐项审核，覆盖以下内容：

- 设计文档全部17个章节及附录（共2066行）
- 2016版规范全文（含30个术语、34个缩略语、8个安全子章节、附录A-P）
- 2022版规范全文（含24个术语、24个缩略语、8个安全子章节、附录A-O）
- 2022版前言中列出的34项变更

### 1.2 审计结论

| 指标 | 数值 |
|------|------|
| 审计方法数量 | 120种 |
| 审计轮次 | 10轮 |
| 发现问题总数 | 28个 |
| 严重问题 | 5个 |
| 中等问题 | 12个 |
| 轻微问题 | 11个 |
| 问题确认率 | 100%（所有问题均经规范原文验证） |

### 1.3 总体评价

设计文档整体结构完整，覆盖了2022版前言中列出的34项变更，对主要变更点（如H.265/AAC编解码扩展、SDP协议引用更新、安全要求重构、注册重定向、设备软件升级等）进行了详细分析。但在以下方面存在不足：

1. **缩略语变更分析不完整**：仅识别了SVAC一个缩略语描述变更，遗漏了14个缩略语的描述变更。
2. **引用标准文件变更严重不完整**：附录A遗漏了大量新增、更新和删除的引用标准。
3. **协议格式变更遗漏**：未提及MANSCDP命令格式中CmdType→cmdType、Message→MESSAGE等系统性大小写变更。
4. **术语变更遗漏**：未提及"压缩编码"→"编码"、"球机/云台"→"摄像机云台"等多个术语变更。
5. **章节引用不精确**：部分章节号引用与规范实际章节号存在偏差。

#### S1-003 [严重] MANSCDP/SIP方法名大小写系统性变更未提及

**审计方法**: A08协议字段格式原文核对法、D03术语一致性验证法、E02信令协议规范验证法

**问题描述**:
2022版对MANSCDP命令格式和SIP方法名进行了系统性大小写调整，设计文档未提及这些变更：

| 序号 | 变更项 | 2016版 | 2022版 | 影响范围 |
|------|--------|--------|--------|----------|
| 1 | XML元素名 | CmdType | cmdType | 全部MANSCDP命令 |
| 2 | SIP方法名 | Message | MESSAGE | 全部MESSAGE方法引用 |
| 3 | SIP方法名 | Info | INFO | 历史媒体回放控制 |

**具体证据**:

2016版9.1节（行283）："前端设备控制、信息查询、报警事件通知和分发等应用的会话控制采用SIP扩展协议IETF RFC 3428规定的**Message**方法实现。"

2022版4.3.1节（行248）："前端设备控制、信息查询、报警事件通知和分发等应用的会话控制采用SIP扩展协议IETF RFC 3428规定的**MESSAGE**方法实现。"

2016版附录A（行548）："设备控制命令应包括命令类型(**CmdType**）、命令序列号(SN)…"

2022版附录A（行531）："设备控制命令应包括命令类型(**cmdType**）、命令序列号(SN)…"

**规范来源**:
- 2016版：28181_2016_full.txt 行283、291、293、548、577
- 2022版：28181_2022_full.txt 行248、255、257、531、570

**设计文档位置**: 未提及

**影响评估**: 严重。XML元素名大小写变更直接影响XML解析和生成代码，SIP方法名大小写变更影响SIP消息构造。这些是协议实现层面的系统性变更，遗漏可能导致互操作性问题。

#### S1-004 [严重] 媒体存储封装格式ISO/IEC 13818-1版本变更遗漏

**审计方法**: A10数值参数原文核对法、A17媒体编解码原文核对法、C06引用准确性验证法

**问题描述**:
2022版6.3节将PS格式引用的ISO/IEC 13818-1标准从2000版更新为2019版，设计文档未在任何章节提及此变更。

**规范原文对比**:

"联网系统中，视音频等媒体数据的存储封装格式应为PS格式，格式见**ISO/IEC 13818-1:2000**。"

2022版6.3节（28181_2022_full.txt）：
"联网系统中视音频等媒体数据的存储封装格式应为PS格式，格式见**ISO/IEC 13818-1:2019**。"

**规范来源**:
- 2016版6.3节：28181_2016_full.txt
- 2022版6.3节：28181_2022_full.txt

**设计文档位置**: 未提及（设计文档第6.3节讨论的是SDP f字段，非媒体存储封装格式）

**影响评估**: 严重。ISO/IEC 13818-1从2000版到2019版跨越19年，涉及MPEG-2系统层规范的重大更新，直接影响PS封装实现。

#### S1-005 [严重] 报警通知命令字段从可选变必选未提及

**审计方法**: A08协议字段格式原文核对法、E04命令格式规范验证法、H06影响范围逻辑验证法

**问题描述**:
2022版9.4.3.1节将报警通知命令中的多个字段从"可选项"改为"应包括"（必选），设计文档未提及此变更。

**规范原文对比**:

2016版9.4.3.1节（28181_2016_full.txt 行577）：
"报警事件通知命令应包括命令类型(CmdType)、命令序列号(SN)、设备编码(DeviceID)、报警级别(AlarmPriority)等。**可选项**：报警时间(AlarmTime)、报警方式(AlarmMethod)、经度(Longitude)、纬度(Latitude)、扩展报警类型(AlarmType)、报警类型参数(AlarmTypeParam)。"

2022版9.4.3.1节（28181_2022_full.txt 行570）：
"报警事件通知命令应包括命令类型(cmdType)、命令序列号(SN)、设备编码(DeviceID)、报警级别(AlarmPriority)、报警时间(AlarmTime)、报警方式(AlarmMethod)、经度(Longitude)、纬度(Latitude)等，扩展报警类型(AlarmType)、报警类型参数(AlarmTypeParam)；"

**变更内容**:
- AlarmTime、AlarmMethod、Longitude、Latitude：从"可选项"变为"应包括"（必选）
- AlarmType、AlarmTypeParam：从"可选项"变为"扩展"
- CmdType → cmdType（大小写变更）

**规范来源**:
- 2016版9.4.3.1节：28181_2016_full.txt 行577
- 2022版9.4.3.1节：28181_2022_full.txt 行570

**设计文档位置**: 未提及

**影响评估**: 严重。字段从可选变必选直接影响XML消息生成和解析逻辑，不实现此变更将导致协议不兼容。

#### S1-006 [中等] 附录K和附录L变更类型标注错误

**审计方法**: A04附录编号引用验证法、D05前后一致性验证法

**问题描述**:
设计文档附录映射表将2022版附录K和附录L标注为"更改"，但2022版前言未提及这两个附录的变更，实际仅为重编号移动，内容无变化。

**设计文档原文**（行1312-1320）:
"| 附录K | 媒体流保活机制 | 规范性 | 附录M | **更改**（原附录M移至此处） |"
"| 附录L | Subject头域定义 | 规范性 | 附录K | **更改**（原附录K移至此处） |"

## 六、审计结论与建议

### 6.1 审计结论

设计文档作为GB/T 28181-2016至2022升级开发的指导文档，整体质量良好，覆盖了主要变更点，但在以下方面需要改进：
