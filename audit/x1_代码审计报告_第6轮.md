## 二、P1-严重问题（新增）

### P1-22: SipTlsProperties.java — @PostConstruct validate()在TLS未启用时也抛异常

**文件**: `news/main/java/com/genersoft/iot/vmp/conf/SipTlsProperties.java`

**问题**: `@PostConstruct`标注的`validate()`方法在TLS未启用时也会抛出`IllegalStateException`，导致应用无法启动。

**代码位置**: 第308-309行

```java
@javax.annotation.PostConstruct
public void validate() { if (!isValid()) throw new IllegalStateException("TLS配置无效"); }
```

**分析**: 
- `isValid()`方法检查`keyStore`和`keyStorePassword`是否为空
- 当TLS未启用（`enabled=false`）时，`keyStore`和`keyStorePassword`通常为空
- 此时`isValid()`返回`false`，`validate()`抛出`IllegalStateException`
- Spring的`@PostConstruct`在Bean初始化时调用，异常会导致应用启动失败

**影响**: 
- 如果用户不配置TLS（大多数场景），应用将无法启动
- 这与TLS作为可选功能的定位矛盾
- 生产环境中将导致服务无法启动

**修复建议**: 在`validate()`中先检查`enabled`是否为`true`，只有启用TLS时才验证配置：
```java
@PostConstruct
public void validate() {
    if (enabled && !isValid()) {
        throw new IllegalStateException("TLS配置无效: 启用TLS时必须提供密钥库和密码");
    }
}
```

### P2-43: SipTlsProperties.java — 密码以明文String存储

**文件**: `news/main/java/com/genersoft/iot/vmp/conf/SipTlsProperties.java`

**问题**: 密钥库密码和信任库密码以明文`String`存储：
- 第66行：`private String keyStorePassword;`
- 第87行：`private String trustStorePassword;`

**分析**: 
- Java的`String`是不可变的，密码在JVM字符串池中驻留，无法主动清除
- 内存dump攻击可以获取明文密码
- 安全最佳实践建议使用`char[]`并在使用后通过`Arrays.fill(password, '\0')`清除

**影响**: 密码可能通过内存dump泄露，降低系统安全性。

**修复建议**: 将密码字段改为`char[]`，并在使用后清除。

### P2-44: SnapshotConfigMessageHandler.java — XML元素命名使用非标准混合大小写

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/transmit/event/request/impl/message/control/cmd/SnapshotConfigMessageHandler.java`

**问题**: XML元素命名使用非标准混合大小写：
- 第116行：`ELEMENT_UPLOAD_URL = "uploaduRL"` — u小写、R大写、L大写
- 第122行：`ELEMENT_SESSION_ID = "sessionID"` — s小写、D大写

**分析**: 
- "uploaduRL"的命名非常不寻常，不符合任何标准命名规范
- 代码注释声称"规范 XML 元素名为 uploaduRL（u 小写）"
- 同样的命名模式出现在`SIPCommander2022Supplement.java`中的"FileuRL"和"FirmWare"
- 这些命名需要与GB28181-2022标准原文核实

**影响**: 
- 如果命名与标准不符，设备将无法正确解析XML
- 命名混淆可能导致与第三方设备的互操作性问题

**修复建议**: 核实GB28181-2022标准原文中的XML元素命名，如与代码不一致则修正。
