### P0-04: DigestServerAuthenticationHelper sm3MessageDigest 实例字段线程不安全（安全漏洞）

**文件**: `patches/02-DigestServerAuthenticationHelper.patch`
**行号**: 第12行（新增字段）、第58-72行（使用处）
**问题**: `sm3MessageDigest` 是 `MessageDigest` 类型的实例字段，被 `DigestServerAuthenticationHelper` 单例 Bean 的所有线程共享。`MessageDigest` **不是线程安全的**，多线程并发调用 `digest()` 会导致数据错乱，可能造成认证绕过。

**问题代码**:
```java
+    private MessageDigest sm3MessageDigest;  // ← 实例字段，多线程共享

// 使用处（多线程并发调用）：
+    byte[] sm3bytes = sm3MessageDigest.digest(A1.getBytes(...));  // ← 线程不安全
+    sm3bytes = sm3MessageDigest.digest(A2.getBytes(...));          // ← 线程不安全
+    sm3bytes = sm3MessageDigest.digest(SM3_KD.getBytes(...));      // ← 线程不安全
```

**影响**: 在高并发注册场景下，SM3 摘要计算可能产生错误结果，导致：
1. 合法设备认证失败（误拒）
2. 非法设备认证通过（认证绕过，**安全漏洞**）
3. 系统不稳定

**修复建议**: 每次调用时创建新的 `MessageDigest` 实例，或使用 `ThreadLocal<MessageDigest>`，或使用 `synchronized` 保护。

### P1-02: SM3DigestHelper SM3 回退到 SHA-256 破坏认证（功能缺陷）

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/auth/SM3DigestHelper.java`
**行号**: 第60-75行（`digest` 方法）
**问题**: 当 SM3 算法不可用时，回退使用 SHA-256。SHA-256 与 SM3 产生不同的摘要值，设备使用 SM3 计算的响应将无法匹配 SHA-256 计算的摘要，导致认证失败。更严重的是，这种回退是静默的，不抛出异常。

**问题代码**:
```java
} catch (NoSuchAlgorithmException e) {
    logger.warn("SM3 algorithm not available, falling back to SHA-256");
    MessageDigest sha256 = MessageDigest.getInstance("SHA-256");  // ← 静默回退，破坏认证
    return bytesToHex(sha256.digest(data));
}
```

**影响**: 若运行环境未注册 BouncyCastle SM3 Provider，所有 SM3 认证将静默回退到 SHA-256，导致设备认证失败，且难以排查。
**修复建议**: SM3 不可用时应抛出异常而非静默回退，或在启动时强制检查 SM3 可用性。

### P1-03: SM3DigestHelper 摘要比较使用 equalsIgnoreCase 存在时序攻击（安全）

**文件**: `news/main/java/com/genersoft/iot/vmp/gb28181/auth/SM3DigestHelper.java`
**行号**: 第90行（`verify` 方法）
**问题**: 摘要比较使用 `equalsIgnoreCase`，该方法是短路比较（发现不匹配字符立即返回），存在时序攻击（Timing Attack）风险。

**问题代码**:
```java
return calculated.equalsIgnoreCase(expected);  // ← 时序攻击风险
```

**影响**: 攻击者可通过测量响应时间逐字节猜测摘要值。
**修复建议**: 使用 `MessageDigest.isEqual()` 进行常量时间比较。

### P1-16: PTZ 标签大小写不一致（合规性）

**文件**: `backend/src/main/java/com/genersoft/iot/vmp/gb28181/transmit/cmd/impl/SIPCommander2022Supplement.java`
**行号**: 第86-88行
**问题**: PTZ 精准控制命令 XML 中，`<pan>` 和 `<zoom>` 使用小写，而 `<Tilt>` 使用大写。响应处理 `PtzPreciseStatusQueryMessageHandler` 中统一使用 `<Pan>`、`<Tilt>`、`<Zoom>`（大写）。标签大小写不一致。

**问题代码**:
```java
xml.append("<pan>").append(...).append("</pan>\r\n");    // ← 小写
xml.append("<Tilt>").append(...).append("</Tilt>\r\n");  // ← 大写
xml.append("<zoom>").append(...).append("</zoom>\r\n");  // ← 小写
```

**影响**: 部分设备可能对 XML 标签大小写敏感，导致 PTZ 精准控制命令解析失败。
**修复建议**: 统一使用 GB28181-2022 规范定义的标签名（通常为大写 `<Pan>`、`<Tilt>`、`<Zoom>`）。
