## 二、P1-严重问题（新增1个）

### R3-P1-01: DeviceControlTypeTest固化可能错误的枚举值

- **文件**：`tests/java/unit/DeviceControlTypeTest.java` 第21, 33, 50行
- **审计方法**：#107 测试断言有效性、#90 XML元素名大小写合规性
- **问题描述**：
  ```java
  assertEquals("PTzPrecisectrl", DeviceControlType.PTZ_PRECISE.getVal());  // 小写z
  assertEquals("FormatsDcard", DeviceControlType.FORMAT_SDCARD.getVal());  // 大写S
  assertEquals("Deviceupgrade", DeviceControlType.DEVICE_UPGRADE.getVal());  // 小写u
  ```
  测试断言固化了大小写混合的枚举值。这些命名需要与GB/T 28181-2022规范原文逐一核对。如果规范使用不同的大小写，测试固化了错误。
- **影响**：如果枚举值与规范不符，设备无法识别命令，且测试无法发现错误。
- **修复建议**：逐项核对规范原文，确认每个枚举值的大小写。

## 三、P2-一般问题（新增3个）

### R3-P2-01: SIPp场景使用硬编码IP和端口

- **文件**：`tests/sipp/register/reg_redirect_302.xml` 第40, 51, 54, 73, 76行
- **审计方法**：#110 测试隔离性
- **问题描述**：SIPp场景使用硬编码的 `10.0.0.100:5060` 等地址，无法适应不同测试环境。
- **修复建议**：使用SIPp的 `[remote_ip]` 和 `[remote_port]` 变量。

### R3-P2-02: SIPp场景SN全部硬编码为1

- **文件**：多个SIPp场景文件
- **审计方法**：#108 边界测试
- **问题描述**：所有测试用例使用 `<SN>1</SN>`，无法验证SN递增逻辑和SN冲突处理。
- **修复建议**：使用递增SN或随机SN。

### R3-P2-03: run_sipp_tests.sh丢弃stderr输出

- **文件**：`tests/scripts/run_sipp_tests.sh` 第22行
- **审计方法**：#112 测试脚本健壮性
- **问题描述**：`2>/dev/null` 丢弃了sipp的错误输出，调试困难。
- **修复建议**：重定向到日志文件而非/dev/null。
