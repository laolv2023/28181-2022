> **核查对象**：WVP-GB28181-pro（wvp-GB28181-pro 仓库，版本 2.7.4）

> **核查方法**：逐项比对 WVP 源码实现与升级开发设计文档要求，定位需要升级的代码模块
> **核查依据**：反幻觉原则——所有数据均来自升级开发设计文档和 WVP 源码，不推测、不补全、不编造；无记录则不声称

## 一、核查概述

### 1.1 WVP 项目概况

WVP-GB28181-pro 是一个基于 GB28181-2016、部标 808、部标 1078 标准实现的开箱即用的网络视频平台，负责实现核心信令与设备管理后台部分。

| 项目 | 说明 |
|------|------|
| 项目名称 | web video platform (WVP) |
| 版本 | 2.7.4 |
| 基础规范 | GB28181-2016 |
| 开发语言 | Java |
| 框架 | Spring Boot |
| 流媒体服务 | ZLMediaKit |
| SIP 协议栈 | JAIN-SIP |
| 源码位置 | `src/main/java/com/genersoft/iot/vmp/` |

**来源**：WVP 仓库 `pom.xml`（version 2.7.4）、`README.md`（基于GB28181-2016）。

### 1.2 核查范围

本报告依据《升级开发设计文档》的 16 个章节内容，对 WVP 的以下模块进行逐项核查：

2. 互联结构（第3章）
3. 通信协议结构（第4章）
4. 媒体传输要求（第5章）
5. 媒体编解码（第6章）
6. SDP 协议（第7章）
7. 安全性要求（第8章）
8. 编码规则（第8章）
9. 注册和注销（第9章）
10. 设备控制（第10章）
11. 设备信息查询（第11章）

15. 兼容性设计（第16章）
16. 升级开发任务清单（第15章）

### 1.3 核查结论摘要

| 类别 | 数量 | 说明 |
|------|------|------|
| **需升级项（高优先级）** | 15 | 影响 2022 版合规性的核心功能缺失或错误 |
| **需升级项（中优先级）** | 11 | 2022 版新增功能未实现（原12项，审核后删除1项无设计文档依据的MANSRTSP协议版本升级项，实际11项） |
| **需升级项（低优先级）** | 8 | 措辞/参数优化项 |
| **字符集与编码规则升级项** | 4 | GB 18030字符集、行政区划代码、类型编码、网络标识 |
| **已合规项** | 18 | WVP 实现符合 2022 版要求 |
| **合计** | 56 | 需升级38项 + 已合规18项 |

**总体评估**：WVP 2.7.4 基于 GB28181-2016 开发，**未实现 2022 版的核心变更**，包括：协议版本标识、SM3 摘要算法、注册重定向、H.265/AAC 编解码协商、新增控制/查询命令、图像抓拍、设备软件升级等。需进行较大规模的升级改造才能达到 2022 版合规。

根据设计文档第15章任务清单和合规性影响程度，建议按以下优先级实施升级：

#### 第一阶段：核心合规性升级（高优先级，预计工作量：大）

1. **SIG-01**: SIP注册消息增加X-GB-ver版本标识头域（附录I）
2. **SIG-05**: SIP信令摘要算法从MD5更新为SM3（8.3）
3. **SIG-02**: 实现注册重定向功能（9.1.2.3）
4. **SDP修复**: s字段DoWnload拼写、删除Talk、a=doWnloadspeed拼写
5. **新增控制命令**: PTZ精准控制/存储卡格式化/目标跟踪/设备软件升级
6. **新增查询命令**: 看守位/巡航轨迹/PTZ精准状态/存储卡状态查询
7. **新增功能**: 图像抓拍、设备软件升级结果通知

#### 第二阶段：媒体编解码升级（中优先级，预计工作量：大）

1. **MED-01**: H.265 RTP载荷类型PT=99→100
2. **MED-02**: AAC音频流RTP封装/解包支持
3. **SDP扩展**: f字段启用、a=filesize/ssvcratio/streamnumber参数
4. **SDP错误响应**: 488/486处理
5. **MANSRTSP升级**: 协议版本、scale取值、倒放命令
6. **TCP媒体传输重连机制**

#### 第三阶段：安全与兼容性升级（低优先级，预计工作量：中）

1. **字符集升级**: GB 2312→GB 18030
2. **Monitor-user-Identity头域**
3. **GB 35114高安全级别**
4. **X-Routepath路径选择**
5. **类型编码/网络标识校验**

### 5.2 升级风险评估

| 风险项 | 风险等级 | 说明 |
|--------|----------|------|
| SM3算法升级 | 高 | 可能导致已注册设备认证失败，需保留MD5兼容 |
| s字段Talk删除 | 中 | 可能影响语音对讲功能，需评估设备兼容性 |

| 字符集升级 | 低 | GB 18030兼容GB 2312，风险较小 |

## 七、核查结论

WVP-GB28181-pro 2.7.4 基于 GB28181-2016 开发，**未实现 2022 版的核心变更**。经逐项核查，共发现 **38 个需升级项**（15个高优先级 + 11个中优先级 + 8个低优先级 + 4个字符集/编码规则升级项），以及 18 个已合规项，合计 56 项。

**核心差距**：
1. **安全性**：SIP信令认证仍使用MD5，未升级为SM3
2. **协议版本**：未实现X-GB-ver版本标识和注册重定向
3. **新增功能**：未实现PTZ精准控制、存储卡格式化、目标跟踪、设备软件升级、图像抓拍等2022版新增功能
4. **新增查询**：未实现看守位/巡航轨迹/PTZ精准状态/存储卡状态查询
5. **SDP协议**：s字段Talk未删除、DoWnload拼写错误、f字段视频点播场景未启用（语音对讲已启用）、新增a字段参数未实现
6. **媒体编解码**：H.265 PT值不符、AAC未实现
7. **字符集**：仍使用GB 2312，未升级为GB 18030

**建议**：WVP 需进行较大规模的升级改造才能达到 GB/T 28181-2022 合规要求。建议按照第五节的升级优先级排序，分三个阶段实施升级，并在每个阶段完成后进行充分的兼容性测试。
